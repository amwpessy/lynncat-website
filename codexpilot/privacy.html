Privacy Policy隐私政策

Local-first by design.以本地优先为设计原则。

CodexPilot is designed for Mac users who want to monitor Codex usage, system health, disk activity, and runtime cache behavior without sending personal monitoring data to a third-party analytics service.CodexPilot 面向在 Mac 上使用 Codex 的用户，用于监控 Codex 额度、系统健康、硬盘读写与运行缓存行为，不会把个人监控数据发送到第三方分析服务。

Data processed on your Mac在 Mac 本地处理的数据

The app may read system status such as CPU, memory, battery, disk capacity, disk read/write activity, and cache size estimates. It may also read user-authorized Codex local log folders to infer quota-related signals when those logs contain usable information.应用可能读取 CPU、内存、电池、硬盘容量、硬盘读写活动与缓存体积估算等系统状态。若用户授权 Codex 本地日志目录，应用也可能读取这些日志，用于推断可用的额度相关信号。

No tracking or advertising profile不做追踪或广告画像

  • CodexPilot does not include third-party advertising SDKs.CodexPilot 不包含第三方广告 SDK。
  • CodexPilot does not build a cross-app advertising profile.CodexPilot 不建立跨应用广告画像。
  • CodexPilot does not sell personal information.CodexPilot 不出售个人信息。

Codex quota visibilityCodex 额度可见性

Quota information depends on what the installed Codex app exposes locally. If Codex changes its local log format or does not expose a value, CodexPilot may show the field as unavailable instead of fabricating data.额度信息取决于当前安装的 Codex 在本地暴露的数据。如果 Codex 更改本地日志格式，或没有暴露某个字段，CodexPilot 会显示不可用，而不是伪造数据。

Cache cleanup缓存清理

Cache cleanup is limited to user-visible or user-authorized cache locations used by CodexPilot's cleanup workflow. The app should ask before destructive cleanup actions and should not delete unrelated user documents.缓存清理仅限 CodexPilot 清理流程识别到的用户可见或用户授权缓存位置。应用应在执行破坏性清理前请求确认，不应删除无关用户文档。
