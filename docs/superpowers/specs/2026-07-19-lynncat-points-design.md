# Lynncat 账号、在线积分与排行榜设计规格

## 目标

为 Lynncat Markets 的 macOS、iOS 与 watchOS 客户端增加统一的 Lynncat 账号和积分系统：用户使用“通过 Apple 登录”建立账号，客户端处于前台可见状态时每在线满一分钟获得 1 个 Lynncat 积分，发送一条市场交流内容消耗 3 个积分，并可查看按当前余额排序的全平台积分排行榜。

本功能复用现有 `lynncat.com/markets` Cloudflare Worker、D1 和逐品种交流系统。积分必须由服务器记账，客户端只显示服务端结果，不能自行增减余额。

## 已确认的产品规则

- 登录方式：仅支持“通过 Apple 登录”，不接入邮箱验证码或邮件服务。
- 游客模式：不强制登录；游客可查看行情、资讯和交流内容，但不能累计积分或发送交流内容。
- 账号同步：同一个 Apple 用户在 Mac、iPhone 和 Apple Watch 上映射到同一个 Lynncat 账号。
- 在线定义：只有客户端处于前台且界面可见、可交互时才累计；后台、锁屏、待机和熄屏不累计。
- 获取速度：每个设备前台连续在线满 1 分钟获得 1 分，不在进入前台时立即赠送。
- 多设备规则：同一账号的不同设备可以同时累计积分；Mac 和 iPhone 同时前台在线 10 分钟可共获得 20 分。
- 单设备去重：同一设备在同一个计分分钟内最多获得 1 分。
- 断网规则：断网期间不在本地补发积分；网络恢复后从新的连续在线区间开始。
- 留言费用：每发送一条交流内容消耗 3 分。
- 发言间隔：继续保留同一用户在同一品种交流窗口 3 分钟一次的限制。
- 免费操作：查看、刷新、举报和屏蔽不消耗积分。
- Watch 范围：完整支持独立登录、在线积分、昵称、发送、举报和屏蔽；不在主界面恢复留言滚动条。
- 排行依据：按当前剩余积分从高到低排序，消费积分后名次可以下降。
- 排行范围：Mac、iPhone、Apple Watch 共用一个全平台总榜，不建立逐品种榜单。
- 排行隐私：用户默认参与，可在设置中隐藏；公开榜单不显示 Apple 姓名、邮箱、Apple 用户标识或设备信息。

## 技术方案

采用已确认的方案：在现有 Cloudflare Worker 与 D1 上实现 Apple 身份校验、Lynncat 会话、设备注册、在线心跳、积分账本和排行榜。三端客户端使用同一套 HTTP API。

不采用客户端本地余额、CloudKit 余额或第三方账号平台。Keychain 只保存 Lynncat 会话令牌和必要的本机设备标识；D1 是账号、余额和积分流水的唯一事实来源。

## Apple 登录与账号生命周期

### 登录

1. 客户端使用系统提供的“通过 Apple 登录”控件发起授权。
2. 客户端把 Apple `identityToken`、`authorizationCode`、一次性随机数和本机安装标识发送到 Worker。
3. Worker 获取并缓存 Apple 公钥，验证令牌签名、签发方、受众、有效期和随机数。
4. Worker 使用 Apple token endpoint 交换授权码，取得用于后续撤销的凭证；刷新令牌使用服务端密钥加密后保存，不能明文落库。
5. Worker 以经过服务端保护的 Apple `sub` 映射 Lynncat 用户，首次登录时创建账号，之后复用原账号。
6. Worker 注册当前设备并签发高强度随机会话令牌；D1 只保存令牌摘要。
7. 客户端把会话令牌保存到 Keychain，后续积分和交流写操作使用 Bearer 鉴权。

iOS、macOS 和 watchOS 的 App ID 必须配置 Sign in with Apple capability，并在 Apple Developer 后台归入能产生一致用户标识的同一组。Worker 使用受众允许列表验证三个客户端的标识，不能接受任意 `aud`。

### 会话

- 会话具有明确有效期、最后使用时间、设备和撤销状态。
- 退出登录只撤销当前设备会话并清除本机 Keychain，账号和积分保留。
- Apple 凭证被撤销或服务端会话失效时，客户端停止累计并回到游客状态。
- 不把 Apple 身份令牌、授权码或会话明文写入日志。

### 删除账号

设置页提供易发现的“删除 Lynncat 账号”操作，并要求二次确认。删除流程必须：

- 撤销 Apple 登录令牌和全部 Lynncat 会话。
- 删除用户资料、设备、积分余额、积分流水和排行榜记录。
- 删除仍然存在的该账号交流内容及其可归属数据。
- 当前设备立即清除 Keychain 和本地账号缓存；其他设备因会话全部撤销，在下次请求时自动清除并退出。
- 返回明确完成状态，不能只做停用或隐藏。

## 数据模型

### `market_users`

保存 Lynncat 账号：`id`、`public_id`、`apple_subject_hash`、`nickname`、`points_balance`、`points_earned_total`、`leaderboard_visible`、`balance_changed_at`、`status`、`created_at`、`updated_at`、`deleted_at`。

- `apple_subject_hash` 唯一，不能保存可直接公开的 Apple `sub`。
- `public_id` 是可轮换的随机榜单标识，不暴露内部用户 ID。
- `points_balance` 不得小于 0。
- `points_earned_total` 仅用于审计和未来统计，本期排行榜不使用它排序。

### `market_apple_credentials`

保存账号删除时所需的 Apple 撤销凭证：`user_id`、`encrypted_refresh_token`、`token_key_version`、`updated_at`。刷新令牌使用独立的 `APPLE_TOKEN_ENCRYPTION_KEY` 加密；应用日志、管理页面和普通 API 永不返回该字段。

### `market_user_devices`

保存账号设备：`id`、`user_id`、`installation_hash`、`platform`（`macos`/`ios`/`watchos`）、`created_at`、`last_seen_at`、`revoked_at`。

同一用户和安装标识唯一。客户端提交的安装标识必须在服务端做带秘密值的摘要，公开响应不返回摘要。

### `market_user_sessions`

保存会话：`id`、`user_id`、`device_id`、`token_hash`、`created_at`、`last_used_at`、`expires_at`、`revoked_at`。`token_hash` 唯一。

### `market_online_leases`

保存每个设备当前的短期在线租约：`device_id`、`user_id`、`started_at`、`last_heartbeat_at`、`active_seconds`、`lease_version`、`updated_at`。

一个设备只能有一个活动租约。心跳间隔超出容忍窗口后，租约失效并从 0 重新累计；不能利用迟到请求补领离线积分。

### `market_point_ledger`

保存积分流水：`id`、`user_id`、`device_id`、`kind`（`online_credit`/`message_debit`/`admin_adjustment`/`reversal`）、`amount`、`balance_after`、`reference_type`、`reference_id`、`idempotency_key`、`created_at`。

- `idempotency_key` 唯一。
- 获得积分记录为正数，消费记录为负数。
- 管理员调整与冲正只预留数据模型，本期不提供公开操作入口。

现有 `market_messages` 增加可空的 `user_id` 和积分流水关联字段，用于原子扣分、账号删除和审计；旧消息保持兼容，游客不能再新建消息。

## 在线计分协议

### 客户端状态机

三端共用以下逻辑：

1. 已登录且场景进入前台可见状态时，启动全 App 唯一的在线任务。
2. 在线任务约每 20 秒调用一次 `POST /markets/points/heartbeat`；本地单调时钟只负责界面进度，不直接决定余额。
3. Worker 根据连续有效心跳累计服务器观察到的在线时长，每达到 60 秒记 1 分并保留不足一分钟的余量。
4. 响应记分时更新客户端余额；重复响应只按服务端余额覆盖，不能在本地累加。
5. 场景进入后台、界面不可见、锁屏、待机、熄屏、退出登录或会话失效时立即停止心跳，并尽力发送停止事件使租约失效。
6. 网络请求失败时显示离线状态；心跳间隔超过服务端容忍窗口后当前租约失效，不把失败区间排队到以后补领。

### 服务端校验

- 心跳必须具有有效会话，并且设备与会话一致。
- 服务端使用自己的时间计算相邻心跳间隔；过快请求不增加时长，超过容忍窗口则重置租约。
- 每次请求带租约版本和幂等键，拒绝重复、倒序或已完成请求。
- 单次心跳最多记 1 分，不根据长时间请求间隔追补多个积分。
- 余额更新、租约更新和积分流水写入必须处于同一原子操作。
- 响应返回最新余额、是否记分、服务器时间和下一次最早可记分时间。

多设备分别拥有独立在线区间，因此可以叠加；同一设备重复启动任务、快速切换页面或重发请求不能叠加。

### 各平台前台判定

- iOS：仅 `scenePhase == .active` 时累计；控制中心、锁屏、后台和非活动状态停止。
- macOS：应用处于活动状态，且主窗口或菜单栏面板实际显示时累计；仅菜单栏进程常驻、窗口全部关闭时不累计。
- watchOS：Watch App 场景活动且屏幕可交互时累计；手腕放下、熄屏或离开 App 时停止。

## 交流内容扣分

`POST /markets/messages` 改为要求 Lynncat Bearer 会话，不再信任客户端提交的匿名 `clientId` 完成新发言。读取、举报和本地屏蔽仍按现有规则工作；举报者身份使用 Lynncat 用户或兼容的匿名安装身份进行去重。

发言服务端顺序：

1. 校验会话、封禁状态、品种房间、昵称和正文。
2. 检查同一用户在同一房间的 3 分钟间隔。
3. 检查当前余额至少为 3。
4. 在一个原子操作中条件扣除 3 分、写入 `message_debit` 流水并创建消息。
5. 返回消息、最新余额和剩余发言间隔。

任一步失败都不得扣分。客户端为每次发送生成幂等键；超时重试不能重复发布或重复扣分。余额不足返回稳定错误码 `insufficient_points`，并带回当前余额和所需积分。

## 昵称与排行榜

### 昵称

- 账号昵称跨设备同步，并作为排行榜名称和默认交流署名。
- 最多 14 个字符，去除首尾空白和不必要换行。
- 服务端执行与交流内容一致的安全过滤，拒绝联系方式、诈骗引导、仇恨、色情、暴力和冒充官方的名称。
- 未设置昵称时由服务端生成稳定的匿名名称，例如 `Lynncat 4821`；不从 Apple 姓名或邮箱自动生成。
- 留言输入区仍允许用户进入昵称编辑入口，但修改的是账号昵称，不再只保存在单机。

### 全平台总榜

- 新增 `GET /markets/leaderboard`，默认返回当前余额最高的前 100 名。
- 排序主键为 `points_balance DESC`；同分时按最后一次余额变化时间升序，再按内部用户 ID 稳定排序。
- 响应仅包含排名、公开昵称、当前积分和当前用户标记所需的非敏感公开 ID。
- 已登录用户即使未进入前 100 名，也能在响应中看到自己的当前名次和余额。
- `leaderboard_visible = 0` 的用户不进入公开排名；本人设置页显示“已隐藏”，不显示模拟名次。
- 用户默认参与，可在设置页即时关闭或重新开启。
- 榜单允许 30 秒短缓存；带“本人名次”的响应不进入共享缓存。
- 获得积分、发送交流内容或修改可见性后，客户端刷新余额和榜单。

## 客户端界面

### 共通交互

- 设置页增加“Lynncat 账号与积分”：Apple 登录、当前余额、积分明细、排行榜可见性、退出登录和删除账号。
- 交流输入区显示当前积分；点击积分进入全平台排行榜。
- 前台累计时显示简洁的“在线累计中”状态，不使用持续跳动或高频动画。
- 积分不足时保留已输入正文，并提示“发送需要 3 积分，还需在线 N 分钟”。
- 游客点击发送时展示 Apple 登录入口和登录用途说明，不阻止查看其他内容。
- 界面文字同时提供简体中文、繁体中文和英文。

### macOS

- 主窗口和菜单栏面板共享一个账号与积分 Store，不能各自启动心跳。
- 主窗口或菜单栏面板任一处可见且 App 活动时累计；二者同时显示仍只算同一设备。
- 设置页提供完整账号管理和积分流水；排行榜使用紧凑列表。

### iOS

- 使用 `scenePhase` 驱动唯一心跳任务。
- 设置页展示完整账号管理；交流输入区与排行榜沿用现有 iPhone 视觉，不新增主 Tab。
- App 切到后台时立即取消未完成分钟。

### watchOS

- 使用 watchOS 系统 Apple 登录流程，支持独立 Watch App 直接建立和登录账号。
- 具体品种的交流页面顶部显示余额、昵称和在线状态，支持系统听写、手写或键盘输入。
- 支持发送、举报和屏蔽；发送规则与其他端一致。
- 主界面不增加留言滚动条，也不恢复已删除的价格提醒。
- 全 App 只使用一个心跳任务；留言列表限量并按需刷新，后台立即取消网络和动画任务。

## API 边界

新增或调整以下路由：

- `POST /markets/auth/apple`：Apple 登录与设备注册。
- `POST /markets/auth/logout`：撤销当前会话。
- `DELETE /markets/account`：删除账号、关联数据并触发 Apple 令牌撤销。
- `GET /markets/account`：账号、昵称、余额和排行榜设置。
- `PUT /markets/account/profile`：修改昵称和排行榜可见性。
- `GET /markets/points/ledger`：分页读取本人积分流水。
- `POST /markets/points/heartbeat`：建立或续期在线租约并尝试记分。
- `POST /markets/points/heartbeat/stop`：前台结束时尽力使在线租约立即失效。
- `GET /markets/leaderboard`：读取公开总榜及本人名次。
- `POST /markets/messages`：改为登录鉴权并原子扣分。

所有账号、积分和写入响应使用 `Cache-Control: no-store`。公开排行榜主体可短缓存，但不得把某个用户的会话信息或本人名次缓存给其他用户。

## 安全、防刷与隐私

- Apple 身份令牌必须由 Worker 验证，客户端返回成功不等于服务端登录成功。
- Apple token endpoint 使用的 Team ID、Key ID、私钥、客户端 ID 允许列表和令牌加密密钥全部通过 Cloudflare Secret 配置，不写入仓库。
- 会话令牌使用足够随机度，服务端只存摘要，Keychain 之外不持久化明文。
- 登录、心跳、昵称修改、排行榜和发言路由分别设置账号、设备和 IP 摘要限流。
- 设备标识只作为防重复和会话管理用途，不用于广告或跨服务跟踪。
- 客户端提交的时间、余额、平台名和积分变化均不可信。
- 管理日志不包含 Apple token、会话令牌、邮箱、完整设备标识或完整 IP。
- 隐私政策和 App Store 隐私标签补充：用户 ID、设备 ID、用户内容、产品交互、账号与积分用途、保留周期和删除方式。
- 积分仅是应用内参与凭证，不可购买、提现、转让或兑换现金；本期不接入 App 内购买。

## 错误处理

- `invalid_apple_token`：Apple 凭证无效，保持游客状态。
- `session_expired`：停止心跳并提示重新登录。
- `heartbeat_too_early`：不记分，使用服务器返回时间继续当前租约。
- `heartbeat_stale`：旧租约失效，从新的在线区间重新计时。
- `heartbeat_duplicate`：不重复记分，刷新服务端余额。
- `insufficient_points`：保留输入内容并显示所需在线时间。
- `cooldown`：继续显示现有 3 分钟倒计时，不扣积分。
- `nickname_rejected`：保留原昵称并给出中英文可操作提示。
- `account_deleted`：清除本机会话并回到游客状态。
- 普通断网：保留最后一次已确认余额并标记为未同步，不本地补分。

## 数据迁移与兼容

- 先以增量 D1 migration 创建账号和积分表，再为 `market_messages` 增加可空关联字段。
- 现有公开消息继续显示至原本的一小时过期，不为旧匿名消息补建账号或扣分记录。
- 发布新客户端前，Worker 必须同时兼容旧客户端的读取和举报；旧客户端发言在过渡期返回明确的 `login_required` 或最低版本更新提示，不能静默失败。
- 版本门控配置在服务端功能稳定后再决定最低可用版本，不在本功能实施中自动停用已上架版本。

## 测试与验收

### Worker 与 D1 自动化测试

- Apple JWT 签名、签发方、受众、有效期、随机数和公钥轮换。
- 首次登录、重复登录、三平台映射同一账号、不同 Apple 用户隔离。
- 会话创建、过期、退出、撤销和账号删除。
- 单设备连续一分钟加 1 分；提前、重复、并发、倒序、租约过期和重试不重复加分。
- 同一账号不同设备同一分钟可以分别加分。
- 后台区间和断网区间不补发。
- 发言成功原子扣 3 分；余额不足、冷却、审核失败和数据库失败不扣分。
- 发送幂等，超时重试不重复消息或流水。
- 排行榜按当前余额排序、同分稳定、前 100、本人名次、隐藏和重新显示。
- 昵称清理、长度限制、敏感内容和匿名名称。
- 删除账号清除关联数据并撤销会话。

### 客户端测试

- Mac 主窗口、菜单栏面板、窗口关闭、切换应用、锁屏和待机的计时边界。
- iPhone 前台、非活动、后台、锁屏、来电或系统遮挡后的计时边界。
