# itnew 科技资讯网站 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 在 Lynncat.com 内交付独立的 itnew 公开资讯站和管理员审核站，形成“每小时多源采集、30 条批次、人工批量审核、站内文章发布、原文链接”的可验证闭环。

**Architecture:** 继续使用现有 Cloudflare Worker 和静态资源部署，但将 itnew 逻辑拆成独立模块，并绑定专用 `ITNEW_DB` D1 与 `ITNEW_IMAGES` R2。公共 API、管理员 API、采集器和发布器共享明确的服务接口；未经明确许可的来源只发布原创摘要与原文链接，获许可来源才保存清理后的全文和图片。

**Tech Stack:** 原生 HTML/CSS/JavaScript、Cloudflare Workers、D1、R2、Cron Triggers、Web Crypto、Node.js 内置 `node:test`。

## Global Constraints

- 不改动现有 `/news/`、`/markets/`、`/xxxc/` 的页面与 API 语义。
- 不触碰或提交当前用户拥有的未跟踪 `dist/` 目录。
- 不把管理员账号、密码、Cookie 签名密钥、CSRF 值或 IP 原文写入仓库、日志或 API 响应。
- `licensed_full` 必须同时通过来源级和文章级许可检查；其余内容固定走 `summary_link`。
- 每个正文分块的 UTF-8 字节数必须 `<= 409600`，图片二进制只进入 R2。
- 前端与后台是两个独立页面；公开页面不展示后台入口。
- 每完成一个任务都先运行该任务测试，再执行全量回归；失败时使用 `superpowers:systematic-debugging`，完成前使用 `superpowers:verification-before-completion`。
- 远程 D1/R2 创建、Secrets 写入、真实来源联网验证和部署属于外部状态操作，只在用户授权的执行阶段进行；计划阶段不伪造资源 ID 或部署结果。

- [ ] **Step 4: 实现采集器**

`collectNextBatch` must follow this exact order:

```js
export async function collectNextBatch(env, options = {}) {
  const now = options.now ?? Date.now();
  const fetchImpl = options.fetchImpl ?? fetch;
  const trigger = options.trigger === 'manual' ? 'manual' : 'cron';
  assertBindings(env);
  await syncSourceRegistry(env.ITNEW_DB, SOURCE_REGISTRY, now);
  const blocking = await getBlockingBatch(env.ITNEW_DB);
  if (blocking) return { status: 'batch_in_progress', batchId: blocking.id };
  const sources = await listEnabledSources(env.ITNEW_DB);
  const settled = await Promise.allSettled(sources.map((source) => fetchSource(source, { now, fetchImpl })));
  const normalized = await persistSourceResultsAndNormalize(env, sources, settled, now);
  if (normalized.successfulSources === 0) return { status: 'all_sources_failed', candidateCount: 0 };
  const fresh = await removeStoredAndNearDuplicates(env.ITNEW_DB, normalized.candidates);
  const chosen = selectBalancedCandidates(fresh, BATCH_TARGET_SIZE);
  if (chosen.length === 0) return { status: 'no_new_candidates', candidateCount: 0 };
  return createOpenBatch(env, { trigger, now, chosen, sourceWarnings: normalized.warnings });
}
```

Every source fetch uses an abort timeout of 12 seconds. R2 staged bodies use `staged/{sha256}.html`; object creation failure records a warning and downgrades that candidate to `summary_link` rather than losing it.

- [ ] **Step 3: 实现 Web Crypto 会话格式**

Session payload fields are exactly `{ sub, exp, csrf, nonce }`; encode as base64url JSON, sign `payload` with HMAC-SHA-256, and store `payload.signature` in `itnew_admin_session`.

```js
export async function requireAdmin(request, env, now = Date.now()) {
  assertAdminSecrets(env);
  const token = readCookie(request, 'itnew_admin_session');
  const session = await verifySessionToken(token, env.ITNEW_SESSION_SECRET, now);
  if (!session || session.sub !== env.ITNEW_ADMIN_USERNAME) throw new AuthError('authentication_required', 401);
  return session;
}

export async function validateAdminMutation(request, env, now = Date.now()) {
  const session = await requireAdmin(request, env, now);
  const requestUrl = new URL(request.url);
  if (request.headers.get('Origin') !== requestUrl.origin) throw new AuthError('invalid_origin', 403);
  if (!constantTimeEqual(request.headers.get('X-CSRF-Token') || '', session.csrf)) throw new AuthError('invalid_csrf', 403);
  return session;
}
```

IP hashing is `SHA-256(ITNEW_IP_PEPPER + ':' + CF-Connecting-IP)`; only digest is stored. Successful login resets the active failure window.
